## Skill trigger: `apm-authoring`

When editing any APM primitive (`apm.yml`, `*.instructions.md`, `SKILL.md`, `*.prompt.md`, `*.agent.md`, hooks, `AGENTS.md`, `CLAUDE.md`), apply the `apm-authoring` skill.

In a repo that uses APM, do not edit `AGENTS.md`, `CLAUDE.md`, or deployed skill files directly — they are regenerated by `apm install` / `apm compile`. Edit the matching primitive under `.apm/` and re-run `apm compile`.
