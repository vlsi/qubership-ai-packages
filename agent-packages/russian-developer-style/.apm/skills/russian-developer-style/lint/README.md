# Lint-конфиги

Стартовые конфиги для механических проверок русского developer-текста.
Конфиги намеренно консервативные: их задача — закрыть базовые правила и
не подавать ложные срабатывания на коде в прозе.

## Что чем покрыто

| Проверка | Инструмент | Конфиг |
| --- | --- | --- |
| орфография, словарь, политика ё/е | `yaspeller` | [`.yaspellerrc.json`](.yaspellerrc.json) |
| грамматика, согласования, повторы | `LanguageTool` (профиль `ru-RU`) | [`languagetool.cfg`](languagetool.cfg) |
| кавычки, тире, неразрывные пробелы, пробел между числом и единицей | `typograf.js` | [`typograf.config.js`](typograf.config.js) |
| продуктовая терминология / запрещённые переводы | свой скрипт + [`../glossary.md`](../glossary.md) | — |
| ICU MessageFormat / Fluent: формы плюрала | свой скрипт | — |
| placeholders в форматных строках (`{0}`, `%s`, `{name}`) | свой скрипт | — |

## Соглашения

- Конфиги покрывают Markdown-файлы и обычный русский текст. Не пропускайте через них
  исходники кода: разбор `*.go`/`*.ts`/`*.java` сделает свои инструменты, а здесь будет шум.
- Названия продуктов и идентификаторы кода должны быть в `dictionary`/allowlist каждого
  инструмента. Иначе мы будем чинить «ошибки» вида `Kubernetes`, `apm`, `Fiber`.
- При работе с CI запускайте линтеры на изменённых файлах, а не на всём репозитории —
  это меняет восприятие ложноположительных срабатываний.

## Локальный прогон

```sh
# yaspeller
npx yaspeller --config .apm/skills/russian-developer-style/lint/.yaspellerrc.json .

# LanguageTool (через CLI или docker image)
languagetool -c .apm/skills/russian-developer-style/lint/languagetool.cfg path/to/file.md

# typograf
node .apm/skills/russian-developer-style/lint/typograf.config.js path/to/file.md
```

Точные команды зависят от того, как вы устанавливаете LanguageTool и
typograf в проекте. Адаптируйте под свой CI и `package.json`/`Makefile`.

## Важное про апстрим

- **yaspeller**: основной репозиторий заархивирован. Используйте свой
  форк или зафиксируйте версию (`yaspeller@8.x`); словарь продуктовых
  терминов держите в репо.
- **LanguageTool**: бесплатный сервер ловит большую часть правил, но
  для CI стоит держать локальный/контейнерный инстанс. Версии правил
  меняются — фиксируйте версию контейнера.
- **typograf.js**: API стабильное, но конкретные имена правил отличаются
  между релизами. Прежде чем разворачивать в CI, прогоните на десятке
  файлов и сверьте результат.
