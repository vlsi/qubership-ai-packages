# Russian developer style: research

Четырёхшаговый редакционный пайплайн, по которому собран APM-пакет
`russian-developer-style`. Сам пакет лежит в
[`agent-packages/russian-developer-style/`](../../agent-packages/russian-developer-style/);
эта папка — журнал работ за ним.

## Метод

| Шаг | Артефакты | Что произошло |
| --- | --- | --- |
| Фаза 1: широкий обзор | `phase1_prompt.md`, `phase1_result.md` | Deep-research-промпт собирает кандидатов в источники стиля. Дёшево, широко, без синтеза. Вывод: готового русского LLM-скилла под технические тексты нет. |
| Фаза 2: глубокая оценка | `phase2_prompt.md`, `phase2_result.md` | Шорт-лист переоценивается по покрытию, тону, риску ложных срабатываний и пересечениям. Вывод: ни один источник не закрывает задачу в одиночку — нужен синтез из Контур.Гайды, Microsoft RU L10n, Ozon Tech styleguide, Mozilla L10n, Норы Галь и корпуса LLM-tells. |
| Фаза 3: синтез | `phase3_prompt.md` → [`SKILL.md`](../../agent-packages/russian-developer-style/.apm/skills/russian-developer-style/SKILL.md) | По результатам фазы 2 собирается сам скилл. Канонический вариант лежит в пакете, не дублируется здесь. |

Двухфазное разделение оставляет глубокую оценку дешёвой за счёт
предварительного отсева кандидатов и сохраняет переиспользуемый
шорт-лист, если бриф потом поменяется.

## Файлы

```text
phase1_prompt.md, phase1_result.md     шорт-лист источников стиля
phase2_prompt.md, phase2_result.md     глубокая оценка, решение про синтез
phase3_prompt.md                       промпт, по которому собран скилл
```

Результат третьей фазы живёт в APM-пакете, а не здесь, чтобы избежать
расхождений. Файлы в этой папке заморожены как журнал работ;
редактируйте скилл по его
[каноническому адресу](../../agent-packages/russian-developer-style/.apm/skills/russian-developer-style/SKILL.md).

## История названия

В промптах фазы 2 и фазы 3 скилл называется `ru-tech-prose`. Это
рабочее имя из исследования — канонический слаг теперь
`russian-developer-style`.

Старое название отбрасывалось по двум причинам. Префикс `ru-`
непрозрачен для роутеров и людей за пределами русскоязычной команды:
`russian-` читается одинаково в любом инструменте. Слово `prose`
сужало восприятие: и роутеры, и читатели трактовали его как намёк, что
скилл работает только на длинных текстах — README, дизайн-доках,
многоабзацных описаниях PR — и пропускали его для однострочных
сообщений об ошибках, трёхсловных подписей кнопок и коротких
`msgstr`-записей в `.po`-файлах. Скилл применяется к русскому
developer-тексту любой длины; новое имя это отражает.

Если будете перезапускать фазовые промпты, считайте слаг
`ru-tech-prose` внутри них артефактом исходного прогона и подставляйте
в новый результат `russian-developer-style`.

## Воспроизведение для другого языка

Пайплайн нейтрален к языку: шаги — обзор, оценка, синтез — не зависят
от того, на каком языке пишется конечный скилл. Большая часть
редакционных правил переживёт смену языка почти без изменений; работа
концентрируется в источниках стиля, типографике, орфографии и
терминологии.

1. Используйте `phase1_prompt.md` и `phase2_prompt.md` как шаблон —
   замените целевой язык, локализационные гайды и кандидатов в
   источники. Структура промптов остаётся.
1. Перепрочитайте таблицу кандидатов из фазы 2; повторно запускайте
   фазу 2 целиком только если для нового языка появляется источник,
   которого не было в русском шорт-листе.
1. Перезапустите фазу 3 с новым языком по умолчанию и подменой
   разделов про типографику, кавычки, тире, неразрывные пробелы и
   ё/е-подобные диалектные особенности.
